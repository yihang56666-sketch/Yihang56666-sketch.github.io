项目概述

这是一个面向电赛送药小车题目的控制系统模板，主控侧基于 TI C2000 F28069，围绕“自动巡线、识别站点、装卸药品、按任务返回”来组织代码。项目不是单个 main.c，而是按固件、视觉和调试工具拆成三层，后续维护和扩展都更清楚。

固件分层

Firmware 目录按照嵌入式工程常见职责拆分：

目录 作用
Config 引脚、板级参数、系统参数
Driver 电机、编码器、红外阵列、串口、RFID、舵机、显示
Module PID、巡线、视觉协议、调度器、站点检测
Algorithm 滤波、融合等算法基础件
App 主循环、命令处理、任务状态机
BSP 板级初始化
Doc 硬件接线和调参说明

任务状态机

送药任务按状态机拆解，避免把业务逻辑堆在循环里：

去程会记录关键路口的转向，返程根据路径记录反向返回。这个设计适合在赛场调试中逐段验证：先跑巡线，再接 RFID，再接舵机装卸，最后合并任务流程。

调试工具

项目里还有 Tool\Python：

  • pid_tuner.py：用于 PID 参数调试
  • serial_plotter.py：串口数据可视化
  • vision_protocol.py：视觉协议封装

这些工具让调参不只停留在串口打印，而是可以把曲线和协议数据接出来看。下一步适合把工具截图、串口日志和调参前后效果补到这篇文章里。

视觉协同

Vision\src 提供视觉侧原型：

  • line_detect.py：线检测逻辑
  • protocol.py：与主控通信的数据格式
  • main.py：视觉主入口

主控侧 Module\vision_protocol.c 负责解析视觉数据，固件和视觉模块之间有明确边界。这样后续无论换 OpenMV、普通 USB 摄像头，还是把算法移到上位机，都不会直接污染主控任务状态机。

下一步优化

  1. 补实车照片、赛道照片和关键接线图。
  2. 给 Firmware 补构建说明，标明 CCS/C2000 编译环境。
  3. 把 PID 调参过程整理成一组“现象 -> 参数 -> 曲线变化”。
  4. 检查代码和文档里是否有不适合公开的比赛资料，再决定是否上传仓库。
