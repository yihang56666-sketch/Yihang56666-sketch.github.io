项目概述

本项目是一个 绳驱巡检装置 的视觉子系统，基于 OpenMV Cam 实现。参赛装置通过绳索悬挂运动，OpenMV 负责识别地面上的橙色圆点标记，为控制板提供末端精确定位修正，并在巡检过程中检测火源。

系统架构

硬件组成

组件 型号 作用
摄像头 OpenMV Cam 图像采集与视觉处理
下位机 自制控制板 电机驱动与运动控制
IMU MPU6050 姿态监测（俯仰/横滚）
通信 UART @ 460800bps OpenMV ↔ 控制板 双向通信

核心算法

1. 橙色目标识别与追踪

通过 LAB 颜色阈值分割提取橙色圆点，配合多级筛选逻辑：

PatrolTargetLock 类实现了断线重锁：当目标短暂丢失时，基于最近位置预测和最大跳变距离容忍，在 2 帧内可自动重锁。

2. 控制确认门 (TargetConfirmGate)

连续 3 帧目标位置稳定，才判定为真正到达，有效防止电机抖动引起的误报。

3. 位置反推

4. MPU6050 姿态监测

I2C 自动寻址，互补滤波融合加速度计与陀螺仪数据，实时监测装置俯仰和横滚角，确保运行姿态稳定。

5. 火源检测

在 SNAKE 模式下，每帧额外检测红色/橙色火源区域，发现后立即上报坐标并暂停运动，等待控制板处理。

UART 通信协议

OpenMV 与控制板之间的双向协议：

方向 命令 说明
控制板 → OpenMV MODE, TARGET 模式切换、目标设定
OpenMV → 控制板 OFFSET 像素偏移量 (dx, dy)
OpenMV → 控制板 ARRIVED 到达目标点通知
OpenMV → 控制板 POSITION 当前世界坐标 (xc, yc)
OpenMV → 控制板 FIRE 火源坐标上报
OpenMV → 控制板 ANGLES 当前俯仰/横滚角

项目特点

  • 全中文算法文档 — 算法详解.md 逐函数解释设计意图与优化理由，方便团队成员理解和维护
  • 生产级细节 — LAB 阈值自动归一化（防写反）、欧氏距离平方代替开根号（嵌入式优化）、到达防抖确认
  • 双通道定位 — 控制板粗定位 + 视觉精修，分担系统压力
  • 一键现场调试 — IDE 实时叠加显示位置、模式、姿态、FPS，配合联调检查表快速排障

该项目是 2026 年校赛”简易绳驱巡检装置”题目的视觉部分实现，完整实现了从橙色目标识别、位置解算、多模式巡检到火源检测的全部功能链路。
